# Messenger Lite

Готовый минимальный мессенджер на **FastAPI + WebSocket + SQLite**.

Что уже умеет:
- регистрация / вход **только по никнейму**;
- список пользователей;
- создание **личного диалога 1-на-1**;
- отправка сообщений в личку;
- история сообщений;
- получение новых сообщений в реальном времени через WebSocket.

## 1. Установка

### Windows

```bash
py -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### Linux / Ubuntu

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

После запуска открой:

```text
http://127.0.0.1:8000
```

## 2. Как тестировать

### На одном компьютере
Открой два разных браузера или два окна в режиме инкогнито.
Зайди под разными никнеймами и пиши друг другу.

### С другого устройства в одной сети
Запускай так:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

Узнай IP компьютера и открой на втором устройстве:

```text
http://ТВОЙ_IP:8000
```

## 4. Как это работает

- Пользователь вводит никнейм.
- Сервер создаёт или обновляет пользователя и выдаёт `session_token`.
- Фронтенд хранит токен в `localStorage`.
- Для обычных запросов токен идёт в заголовке `X-Session-Token`.
- Для real-time фронтенд подключается к `/ws?token=...`.
- Сообщения сохраняются в `chat.db`.

## 5. База данных

Используется SQLite файл:

```text
chat.db
```

Для первого запуска это удобно. Позже можно заменить на PostgreSQL.

## 7. Важно

Это **MVP**. Он подходит для старта, локального теста и первой сборки на сервере, но:
- без паролей;
- без полноценной защиты;
- без масштабирования на несколько backend-инстансов.

Для первого личного мессенджера — нормальная база.
